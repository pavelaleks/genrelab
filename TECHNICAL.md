## 🎯 Обзор проекта

**NARRALAB** — это профессиональная платформа для создания интерактивных историй, анализа литературных жанров и экспериментов с трансмедиальным сторителлингом с использованием искусственного интеллекта.

### Основные возможности:

- **Генерация текстов** в различных литературных жанрах с настраиваемыми параметрами
- **Анализ соответствия** сгенерированных текстов жанровым нормам
- **Литературоведческий анализ** пользовательских текстов
- **Создание ветвящихся нарративов** с интерактивными развилками
- **Трансформация текстов** между различными медиаформатами
- **Визуализация сюжетных структур** в виде интерактивных графов
- **Профилирование жанров** с помощью радарных диаграмм

### Целевая аудитория:

- Писатели и сценаристы
- Контент-мейкеры и блогеры
- Преподаватели литературы и творческого письма
- Исследователи нарративных структур
- Разработчики интерактивных историй

## ⚙️ Функциональность

### 1. Генератор историй (Genre Generator)

#### Описание
Модуль для генерации текстов в различных литературных жанрах с использованием ИИ.

#### Поддерживаемые жанры:

1. **Сказка** (`fairy_tale`)
   - Профиль: высокая сюжетность (85), условность (95), нравственная окраска (90)
   - Особенности: волшебные элементы, архетипические персонажи, дидактичность

2. **Притча** (`parable`)
   - Профиль: низкая описательность (20), высокая нравственная окраска (95)
   - Особенности: краткость, аллегоричность, мораль

3. **Рассказ** (`short_story`)
   - Профиль: сбалансированный профиль
   - Особенности: завершённость, концентрация действия, психологизм

4. **Путевой очерк** (`travelogue`)
   - Профиль: высокая описательность (80), социальность (70)
   - Особенности: документальность, описания мест, личные впечатления

5. **Антиутопический фрагмент** (`dystopian`)
   - Профиль: высокая конфликтность (85), социальность (90)
   - Особенности: критика общества, мрачная атмосфера, предупреждение

6. **Научная заметка** (`scientific_note`)
   - Профиль: низкая лиричность (10), высокая описательность (70)
   - Особенности: объективность, точность, терминология

7. **Дневниковая запись** (`diary_entry`)
   - Профиль: высокая лиричность (80), низкая сюжетность (30)
   - Особенности: субъективность, эмоциональность, хронология

#### Настраиваемые параметры:

| Параметр | Тип | Диапазон/Опции | Описание |
|----------|-----|----------------|----------|
| **Тональность** | selectbox | ироничная, серьёзная, трагическая, философская, сатирическая | Общая эмоциональная окраска текста |
| **Фокус повествования** | selectbox | герой, мир, событие, мысль | На чём концентрируется внимание |
| **Уровень описательности** | slider | 0-100 | Детализация описаний |
| **Количество персонажей** | slider | 1-5 | Число действующих лиц |
| **Наличие моральной развязки** | checkbox | да/нет | Наличие явной морали |
| **Сеттинг/эпоха** | selectbox | современность, XIX век, будущее, и др. | Время и место действия |
| **Объём текста** | slider | 100-400 слов | Приблизительная длина текста |

#### Процесс генерации:

1. **Выбор жанра** — пользователь выбирает жанр из списка
2. **Настройка параметров** — установка характеристик текста
3. **Формирование промпта** — система создаёт детальный промпт на основе:
   - Описания жанра
   - Типичных признаков
   - Структурной схемы
   - Инструкций по генерации
   - Параметров пользователя
4. **Запрос к API** — отправка запроса к Grok API
5. **Проверка завершённости** — автоматическая проверка и догенерация при необходимости
6. **Отображение результата** — показ сгенерированного текста с возможностью редактирования

#### Технические детали:

- **Модель по умолчанию**: `grok-4-fast-reasoning`
- **Temperature**: 0.8 (для творческой генерации)
- **Max tokens**: рассчитывается как `target_length * 2`
- **Автодогенерация**: если текст обрывается, система автоматически завершает его

### 2. Анализ соответствия жанру

#### Описание
Детальный анализ сгенерированного текста на соответствие выбранному жанру.

#### Метрики анализа:

1. **Соответствие жанру** (0-100%)
   - Оценка того, насколько текст соответствует жанровым нормам

2. **Соответствие тональности** (0-100%)
   - Проверка соответствия заданной тональности

3. **Соответствие структуре** (0-100%)
   - Анализ соответствия структурной схеме жанра

4. **Соответствие стилю** (0-100%)
   - Оценка стилистических особенностей

#### Результаты анализа включают:

- **Оценки по метрикам** — числовые значения в процентах
- **Общая оценка** — текстовый комментарий
- **Сильные стороны** — что сделано хорошо
- **Слабые места** — что можно улучшить
- **Рекомендации** — конкретные советы по улучшению
- **Радарная диаграмма** — визуальное сравнение идеального профиля жанра и профиля текста

#### Технические детали:

- Анализ выполняется через LLM (Grok API)
- Используется промпт из `prompts/analyze_prompt.txt`
- Результат парсится из JSON-ответа модели
- Радарная диаграмма строится с помощью Plotly

### 3. Режим "Сломать жанр"

#### Описание
Создание модифицированной версии текста, которая осознанно нарушает жанровые нормы, но сохраняет узнаваемые элементы.

#### Применение:
- Понимание границ жанра через их нарушение
- Экспериментальное творчество
- Обучение жанровым конвенциям

#### Процесс:
1. Система анализирует исходный текст
2. Определяет ключевые жанровые признаки
3. Создаёт версию, которая нарушает эти признаки
4. Сохраняет связь с исходным жанром для узнаваемости

### 4. Анализ пользовательского текста

#### Описание
Полный литературоведческий анализ загруженного или вставленного текста.

#### Возможности анализа:

##### 4.1. Жанровая принадлежность

- **Основной жанр** — определение доминирующего жанра
- **Вероятности жанров** — процентные значения для всех 7 жанров
- **Элементы других жанров** — признаки других жанров в тексте
- **Объяснение** — обоснование жанровой принадлежности

##### 4.2. Структурный анализ

Анализ композиции текста:

- **Тип структуры** — линейная, кольцевая, фрагментарная и т.д.
- **Фазы развития**:
  - Экспозиция
  - Завязка/Конфликт
  - Развитие
  - Кульминация
  - Развязка
- **Комментарий** — общая характеристика структуры

##### 4.3. Нарратив и фокализация

- **Тип повествователя**:
  - От первого лица
  - От третьего лица
  - Всеведущий рассказчик
  - Ненадёжный рассказчик
- **Фокализация**:
  - Нулевая (внешняя)
  - Внутренняя (от персонажа)
  - Внешняя (ограниченная)
- **Временной поток** — линейный, нелинейный, ретроспективный
- **Объяснения** — обоснование каждого аспекта

##### 4.4. Стилистический анализ

- **Регистр** — высокий, средний, низкий
- **Лексические особенности** — специфическая лексика
- **Синтаксические особенности** — структура предложений
- **Риторические приёмы** — метафоры, сравнения, эпитеты и т.д.
- **Общая характеристика** — общее описание стиля

##### 4.5. Цитаты-доказательства

Для каждого аспекта анализа система предоставляет:
- **Цитату из текста** — конкретный пример
- **Объяснение** — как цитата подтверждает анализ

#### Форматы ввода:

- **Загрузка файла** — текстовый файл (.txt)
- **Вставка текста** — прямое введение в текстовое поле

#### Ограничения:

- Минимальная длина: 50 символов
- Рекомендуемая длина: 200-2000 символов для качественного анализа

### 5. Narrative Playground

Экспериментальная площадка для работы с нелинейными нарративами и трансмедиальностью.

#### 5.1. Конструктор сюжета (Plot Builder)

##### Описание
Создание структурированных сюжетных схем с узлами и связями.

##### Типы структур:

1. **Linear** (линейная)
   - Последовательная цепочка событий
   - A → B → C → D

2. **Branching** (ветвящаяся)
   - Развилки сюжета
   - Множественные пути развития

3. **Circular** (циклическая)
   - Замкнутая структура
   - Возврат к началу

4. **Mosaic** (мозаичная)
   - Фрагментарная структура
   - Несколько параллельных линий

5. **Rashomon** (множественные перспективы)
   - Одно событие с разных точек зрения
   - Несколько версий одной истории

6. **Split-perspective** (параллельные линии)
   - Параллельное развитие нескольких сюжетных линий

7. **Epistolary** (эпистолярная)
   - Структура через письма/документы
   - Фрагментарное повествование

##### Параметры генерации:

- **Тип структуры** — выбор из 7 типов
- **Количество узлов** — 3-15 узлов
- **Жанр** — выбор жанра для сюжета
- **Стилистика** — реалистическая, романтическая, модернистская и т.д.
- **Эпоха** — время действия

##### Результат:

- **JSON-структура** с узлами и связями
- **Визуализация графа** — интерактивный граф Plotly
- **Статистика графа**:
  - Количество узлов
  - Количество связей
  - Средняя степень узлов
  - Наличие циклов
- **Генерация текста для узлов** — возможность создать текст для каждого узла

##### Технические детали:

- Используется NetworkX для построения графов
- Plotly для визуализации
- Промпт из `narrative/prompts/plot_prompt.txt`

#### 5.2. Ветвящиеся истории (Branching Narrative Lab)

##### Описание
Создание интерактивных историй с развилками и выбором.

##### Процесс работы:

1. **Начальная сцена** — описание начальной ситуации
2. **Вариант выбора** — описание выбора, который приведёт к ветвлению
3. **Генерация ветвления** — система создаёт альтернативное продолжение
4. **Анализ ветвления** — автоматический анализ изменений:
   - Жанровый сдвиг
   - Изменение фокализации
   - Изменение стиля
   - Изменение напряжения и темпа
5. **История ветвлений** — все ветвления сохраняются и отображаются
6. **Сравнительный анализ** — при наличии нескольких ветвлений система сравнивает их

##### Особенности:

- **Независимые ветвления** — каждое ветвление создаётся отдельно
- **Автоматический анализ** — система анализирует влияние выбора на историю
- **Сравнение** — возможность сравнить разные пути развития

##### Технические детали:

- Промпт из `narrative/prompts/branching_prompt.txt`
- Анализ выполняется автоматически после генерации
- История хранится в `session_state.branching_history`

#### 5.3. Трансформация медиума (Form Shifter)

##### Описание
Преобразование текста между различными медиаформатами.

##### Поддерживаемые форматы:

1. **Пьеса** — диалоги, ремарки, действия
2. **Сценарий** — сценарный формат с описанием кадров
3. **Подкаст** — разговорный формат, диалоги
4. **Комикс** — описание панелей и диалогов
5. **Визуальный роман** — формат для интерактивных визуальных новелл
6. **Игровая сцена** — формат для игровых диалогов
7. **Дневниковая запись** — личный дневник
8. **Соцсетевой пост** — формат для социальных сетей
9. **Поэтическая версия** — стихотворная форма

##### Результат трансформации:

- **Преобразованный текст** — текст в новом формате
- **Анализ изменений**:
  - **Изменение структуры** — как изменилась композиция
  - **Изменение фокализации** — как изменилась точка зрения
  - **Изменение стиля** — как изменился стиль
  - **Новый эффект формата** — какой эффект создаёт новый формат

##### Ограничения:

- Максимальная длина входного текста: 2500 символов
- Валидация длины перед трансформацией

##### Технические детали:

- Промпт из `narrative/prompts/transform_prompt.txt`
- Автоматическая валидация длины текста
- Детальный анализ изменений

#### 5.4. Режим аудитории

##### Описание
Многопользовательский режим для работы в аудитории.

- Используется библиотека `qrcode` для генерации QR-кодов
- QR-код содержит URL для доступа к приложению
- Каждая сессия Streamlit независима

## 🔌 API и интеграции

### Grok API (xAI)

#### Конфигурация:

- **Endpoint**: `https://api.x.ai/v1/chat/completions`
- **Модель по умолчанию**: `grok-4-fast-reasoning`
- **Аутентификация**: Bearer token через `GROK_API_KEY`

#### Использование:

```python
from services.grok_client import call_grok_chat

response = call_grok_chat(
    system_prompt="Ты — опытный писатель.",
    user_prompt="Создай рассказ о...",
    temperature=0.8,
    max_tokens=1000
)
```

#### Обработка ошибок:

- **401 Unauthorized** — неверный или отсутствующий API ключ
- **Timeout** — таймаут 60 секунд
- **JSON parsing** — обработка ошибок парсинга ответа

#### Функции клиента:

- `call_grok_chat()` — основной метод для запросов
- `refine_generation_if_needed()` — догенерация незавершённого текста
- `is_text_complete()` — проверка завершённости текста

## 🔧 Расширение функциональности

### Добавление нового жанра

1. Откройте `genres/schema.py`
2. Добавьте новый объект `Genre` в функцию `get_all_genres()`:

```python
Genre(
    id="new_genre_id",
    name="Название жанра",
    description="Подробное описание...",
    typical_features=["признак 1", "признак 2"],
    structural_schema=["этап 1", "этап 2"],
    radar_profile={
        "сюжетность": 70,
        "описательность": 60,
        "конфликтность": 50,
        "лиричность": 40,
        "условность": 30,
        "нравственная окраска": 60,
        "социальность": 50
    },
    generation_instructions="Инструкции по созданию..."
)
```

3. Убедитесь, что `radar_profile` содержит все 7 осей (0-100)
4. Новый жанр появится в интерфейсе автоматически

### Модификация промптов

Промпты находятся в директории `prompts/`:

- `generate_prompt.txt` — системный промпт для генерации
- `analyze_prompt.txt` — системный промпт для анализа
- `user_analysis_prompt.txt` — промпт для анализа пользовательских текстов

После изменения промптов перезапустите приложение.

### Изменение модели

По умолчанию используется `grok-4-fast-reasoning`. Для изменения:

1. Откройте `services/grok_client.py`
2. Измените `DEFAULT_MODEL` или передайте параметр `model` в `call_grok_chat()`

Доступные модели: см. [документацию xAI](https://docs.x.ai/docs/models)

### Добавление нового формата трансформации

1. Откройте `narrative/transformations.py`
2. Добавьте новый формат в список `TARGET_FORMATS`
3. Обновите промпт в `narrative/prompts/transform_prompt.txt` при необходимости

## 🔍 Технические детали

### Управление состоянием

Приложение использует Streamlit `session_state` для хранения:

- `selected_genre_id` — выбранный жанр
- `generated_text` — сгенерированный текст
- `analysis_result` — результат анализа
- `broken_text` — текст в режиме "сломать жанр"
- `user_text_analysis` — анализ пользовательского текста
- `plot_structure` — структура сюжета
- `branching_history` — история ветвлений
- `transformation_result` — результат трансформации

### Оптимизация запросов

Модуль `narrative/load_balance.py` обеспечивает:

- **Валидацию длины текста** — ограничение до 2500 символов
- **Задержку между запросами** — предотвращение одновременных запросов
- **Обработку ошибок** — graceful обработка ошибок API
- **Защиту от двойных кликов** — блокировка кнопок при выполнении

### Визуализация

- **Радарные диаграммы** — Plotly для профилей жанров
- **Сюжетные графы** — NetworkX + Plotly для визуализации структур
- **QR-коды** — библиотека qrcode для генерации кодов доступа

### Безопасность

- API ключи хранятся в `.env` (не коммитятся в Git)
- `.env` добавлен в `.gitignore`
- Валидация входных данных перед отправкой в API
- Обработка ошибок авторизации

### Производительность

- Асинхронная обработка запросов (где возможно)
- Кэширование результатов (через session_state)
- Оптимизация промптов для уменьшения токенов
- Ограничение длины текста для трансформаций
