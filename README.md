# ✨ NARRALAB

**Платформа интерактивного сторителлинга с использованием ИИ**

NARRALAB — это профессиональная платформа для создания интерактивных историй, анализа литературных жанров и экспериментов с трансмедиальным сторителлингом. Используйте возможности искусственного интеллекта для генерации, анализа и трансформации текстов. Платформа предназначена для писателей, сценаристов, контент-мейкеров и всех, кто работает с текстами и историями.

## 🎯 Функции

### Основные возможности:

1. **Выбор жанра** — выбор из 7 литературных жанров:
   - Сказка
   - Притча
   - Рассказ
   - Путевой очерк (травелог)
   - Антиутопический фрагмент
   - Научная заметка
   - Дневниковая запись

2. **Настройка параметров** — гибкая настройка характеристик текста:
   - Тональность (ироничная, серьёзная, трагическая, философская, сатирическая)
   - Фокус повествования (герой, мир, событие, мысль)
   - Уровень описательности (0-100)
   - Количество персонажей (1-5)
   - Наличие моральной развязки
   - Сеттинг/эпоха
   - Примерный объём текста (100-400 слов)

3. **Генерация текста** — создание текста в выбранном жанре с учётом всех параметров с помощью Grok API

4. **Анализ соответствия жанру** — детальный анализ сгенерированного текста:
   - Оценка соответствия жанру, тональности, структуре и стилю
   - Визуализация профиля текста на радарной диаграмме
   - Конструктивная обратная связь с указанием сильных и слабых сторон
   - Рекомендации по улучшению

5. **Визуализация профиля жанра** — интерактивная радарная диаграмма, показывающая характерные особенности жанра по осям:
   - Сюжетность
   - Описательность
   - Конфликтность
   - Лиричность
   - Условность
   - Нравственная окраска
   - Социальность

6. **Режим «Сломать жанр»** — создание модифицированной версии текста, которая осознанно нарушает жанровые нормы, но сохраняет узнаваемые элементы. Это помогает понять границы жанра через их нарушение.

7. **Анализ пользовательского текста** — полный литературоведческий анализ загруженного или вставленного текста:
   - Жанровая принадлежность с вероятностями для всех жанров
   - Элементы других жанров с указанием признаков
   - Структурный анализ (экспозиция, завязка, развитие, кульминация, развязка)
   - Нарратив (тип повествователя)
   - Фокализация (нулевая, внутренняя, внешняя) с обоснованием
   - Временная организация текста
   - Стилистический анализ (регистр, лексика, синтаксис, риторические приёмы)
   - Доказательные цитаты из текста для каждого аспекта

8. **Narrative Playground** — экспериментальная площадка для изучения трансмедиальности, нелинейного сторителлинга, ветвящихся нарративов и сюжетных структур:
   - **Конструктор сюжета (Plot Builder)**: Создание структурированных сюжетных схем с узлами и связями различных типов (linear, branching, circular, mosaic, Rashomon, split-perspective, epistolary)
   - **Ветвящиеся истории (Branching Narrative Lab)**: Создание ветвящихся историй в простом учебном формате с автоматическим анализом влияния выбора на жанр, фокализацию, стиль и напряжение
   - **Трансформация медиума (Form Shifter)**: Преобразование текста между различными медиаформатами (пьеса, сценарий, подкаст, комикс, визуальный роман, игровая сцена, дневниковая запись, соцсетевой пост, поэтическая версия)
   - **Режим аудитории**: QR-код для доступа к приложению через ngrok, поддержка многопользовательской работы

## 📋 Требования

- **Python** 3.10 или выше
- **GROK_API_KEY** — API ключ от Grok от xAI (можно получить на [console.x.ai](https://console.x.ai))
- **Опционально: ngrok** — для работы в режиме аудитории (установка: `pip install ngrok` или через [ngrok.com](https://ngrok.com))

## 🚀 Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/pavelaleks/genrelab.git
cd genrelab
```

### 2. Создание виртуального окружения

```bash
python -m venv venv
```

### 3. Активация виртуального окружения

**На macOS/Linux:**
```bash
source venv/bin/activate
```

**На Windows:**
```bash
venv\Scripts\activate
```

### 4. Установка зависимостей

```bash
pip install -r requirements.txt
```

**Важно:** Если вы используете виртуальное окружение (`.venv`), убедитесь, что оно активировано перед установкой:

```bash
# Активация виртуального окружения
source .venv/bin/activate  # На macOS/Linux
# или
.venv\Scripts\activate      # На Windows

# Затем установка зависимостей
pip install -r requirements.txt
```

Если после установки всё ещё возникает ошибка `ModuleNotFoundError` для `networkx`, `qrcode` или `plotly`, установите их вручную:

```bash
pip install networkx qrcode[pil] pillow plotly
```

### 5. Настройка переменных окружения

Создайте файл `.env` в корне проекта на основе `.env.example`:

```bash
cp .env.example .env
```

Откройте `.env` и добавьте ваш API ключ:

```
GROK_API_KEY=your_grok_api_key_here
```

## ▶️ Запуск

После установки всех зависимостей и настройки `.env` файла, запустите приложение:

```bash
streamlit run app.py
```

Приложение откроется в браузере по адресу `http://localhost:8501`.

   - Радарную диаграмму с сравнением профилей
   - Комментарии с сильными и слабыми сторонами
   - Рекомендации по улучшению

- Экспериментируйте с разными параметрами для одного жанра, чтобы увидеть, как они влияют на текст
- Сравнивайте профили разных жанров на радарных диаграммах
- Используйте режим «Сломать жанр» для понимания границ жанра
- Анализируйте тексты после редактирования, чтобы увидеть, как изменения влияют на соответствие жанру

## ➕ Как добавлять новые жанры

Чтобы добавить новый жанр:

1. Откройте файл `genres/schema.py`
2. Создайте новый объект `Genre` в функции `get_all_genres()`:

```python
Genre(
    id="new_genre_id",
    name="Название жанра",
    description="Подробное описание жанра...",
    typical_features=["признак 1", "признак 2", ...],
    structural_schema=["этап 1", "этап 2", ...],
    radar_profile={
        "сюжетность": 70,
        "описательность": 60,
        # ... остальные оси
    },
    generation_instructions="Инструкции по созданию текста в этом жанре..."
)
```

3. Убедитесь, что `radar_profile` содержит все 7 осей:
   - сюжетность
   - описательность
   - конфликтность
   - лиричность
   - условность
   - нравственная окраска
   - социальность

4. Значения должны быть в диапазоне 0-100

Новый жанр автоматически появится в интерфейсе при следующем запуске приложения.

## ✏️ Как модифицировать промпты

- **`analyze_prompt.txt`** — системный промпт для анализа текстов. Здесь можно изменить инструкции для модели о том, как анализировать тексты.

## 🎭 Narrative Playground

### Что такое Narrative Playground?

**Narrative Playground** — это экспериментальная площадка для изучения трансмедиальности, нелинейного сторителлинга, ветвящихся нарративов и сюжетных структур. Модуль предназначен для исследования различных нарративных форм, создания ветвящихся историй и экспериментов с трансформацией текста между различными медиаформатами.

**Важно:** Модуль ветвлений работает в простом учебном формате — создаются независимые ветвления без автоматических связей между узлами. Студенты могут собирать связи между ветвлениями вручную в аудитории.

### Основные возможности:

#### 1. Конструктор сюжета (Plot Builder)

Создавайте структурированные сюжетные схемы с узлами и связями:
- **Типы структур**: linear (линейная), branching (ветвящаяся), circular (циклическая), mosaic (мозаичная), Rashomon (множественные перспективы), split-perspective (параллельные линии), epistolary (эпистолярная)
- **Параметры**: количество узлов (3-15), жанр, стилистика, эпоха
- **Визуализация**: интерактивный граф сюжета с помощью Plotly
- **Генерация текста**: создание текста для отдельных узлов сюжета

#### 2. Ветвящиеся истории (Branching Narrative Lab)

Создавайте ветвящиеся истории в простом учебном формате:
- **Начальная сцена**: опишите начальную ситуацию истории
- **Вариант выбора**: добавьте выбор, который приведёт к ветвлению сюжета
- **Генерация ветвления**: система создаёт альтернативное продолжение на основе вашего выбора
- **История ветвлений**: все созданные ветвления сохраняются и отображаются в виде списка
- **Автоматический анализ**: для каждого ветвления система анализирует:
  - Жанровый сдвиг
  - Изменение фокализации (перспективы)
  - Изменение стиля повествования
  - Изменение напряжения и темпа
- **Сравнительный анализ**: при наличии нескольких ветвлений система сравнивает их между собой
- **Простота использования**: модуль работает в учебном формате — студенты создают независимые ветвления, которые можно собирать вручную в аудитории

**Примечание:** Модуль создаёт независимые ветвления без автоматических связей между узлами. Связи можно устанавливать вручную во время работы в аудитории.

#### 3. Трансформация медиума (Form Shifter)

Преобразуйте текст между различными медиаформатами:
- **Доступные форматы**: пьеса, сценарий, подкаст, комикс, визуальный роман, игровая сцена, дневниковая запись, соцсетевой пост, поэтическая версия
- **Детальный анализ**: для каждой трансформации система объясняет:
  - Как изменилась структура текста
  - Как изменилась фокализация (перспектива, точка зрения)
  - Как изменился стиль (регистр, лексика, синтаксис)
  - Какой новый эффект создаёт формат

### Использование в аудитории

**Narrative Playground** поддерживает многопользовательский режим работы:

1. **Запуск через ngrok**:
   ```bash
   # Установите ngrok (если ещё не установлен)
   # Запустите Streamlit приложение
   streamlit run app.py
   
   # В другом терминале запустите ngrok
   ngrok http 8501
   ```

2. **Настройка URL**:
   - Скопируйте URL от ngrok (например, `https://xxx.ngrok.io`)
   - Установите переменную окружения: `export NGROK_URL=https://xxx.ngrok.io`
   - Или введите URL вручную в интерфейсе

3. **QR-код для доступа**:
   - В Narrative Playground нажмите "Режим аудитории"
   - Отсканируйте QR-код с помощью смартфона
   - Студенты получают доступ к приложению на своих устройствах

4. **Независимые сессии**:
   - Каждый студент получает свою независимую сессию Streamlit
   - Все могут работать одновременно без конфликтов
   - Данные не смешиваются между пользователями

### Оптимизации для многопользовательской работы

- **Ограничение длины текста**: максимальная длина вводимого текста — 2500 символов
- **Задержка между запросами**: автоматическая задержка для предотвращения одновременных запросов
- **Обработка ошибок**: graceful обработка ошибок API с понятными сообщениями
- **Защита от двойных кликов**: кнопки блокируются при выполнении запросов

### Примеры использования

#### Пример 1: Создание ветвящейся истории

1. Введите начальную сцену: "Герой стоит на развилке дорог"
2. Добавьте вариант выбора: "Герой решает пойти налево"

6. При наличии нескольких ветвлений система покажет сравнительный анализ
7. Все ветвления сохраняются и отображаются в списке — их можно использовать для создания связей вручную в аудитории

#### Пример 2: Трансформация текста в пьесу

1. Введите исходный текст (например, отрывок из рассказа)
2. Выберите целевой формат: "пьеса"
3. Система преобразует текст, адаптируя структуру под формат пьесы
4. Изучите объяснение изменений: структура, фокализация, стиль, эффект

#### Пример 3: Создание сюжетной структуры

1. Выберите тип структуры: "branching"
2. Установите количество узлов: 7
3. Выберите жанр и параметры
4. Сгенерируйте структуру
5. Визуализируйте граф сюжета
6. Сгенерируйте текст для отдельных узлов

## 🐛 Решение проблем

### Ошибка "ModuleNotFoundError: Библиотека networkx не установлена"

Если вы видите ошибку о том, что библиотека `networkx`, `qrcode` или `plotly` не установлена:

1. **Убедитесь, что виртуальное окружение активировано:**
   ```bash
   source .venv/bin/activate  # macOS/Linux
   # или
   .venv\Scripts\activate    # Windows
   ```

2. **Установите все зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Или установите конкретную библиотеку:**
   ```bash
   pip install networkx qrcode[pil] pillow plotly
   ```

4. **Перезапустите Streamlit приложение:**
   ```bash
   streamlit run app.py
   ```

**Примечание:** Если вы используете PyCharm или другую IDE, убедитесь, что она использует правильное виртуальное окружение (`.venv`).

### Ошибка "GROK_API_KEY не найден"

Убедитесь, что:
- Файл `.env` существует в корне проекта
